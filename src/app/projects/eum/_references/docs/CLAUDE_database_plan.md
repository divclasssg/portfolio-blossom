# 데이터베이스 설계 (계획 단계)

> 아직 확정되지 않은 설계안. 환자 앱 온보딩/회원가입 구현 시 확정 예정.

---

## 배경

- 환자 앱에서 회원가입을 구현할 계획
- 소셜 로그인 없이, 더미 프로필 수준의 정보를 직접 입력받아 저장
- 환자 앱과 의사 패널이 하나의 서버/DB를 공유

---

## 저장소: Supabase (PostgreSQL)

- 무료 티어: 500MB DB, 50K 월간 요청
- 이메일/비밀번호 기본 Auth 내장
- Vercel 배포와 호환 (환경변수 연결)

---

## 데이터 흐름

```
[환자 앱]                    [Supabase DB]                [의사 패널]
회원가입 ──→ patients 테이블
증상 기록 ──→ symptom_records 테이블
                                    ←── 환자 데이터 조회
              ai_briefings 테이블  ←── AI 파이프라인 결과 저장
              ai_suggestions 테이블
                                    ←── AI 결과 조회
              result_packages 테이블 ←── 의사 작성
결과 확인 ←──
```

---

## 기존 JSON 파일의 역할 변경

| 기존 | 변경 후 |
|---|---|
| 런타임 데이터 소스 | DB 스키마 설계의 레퍼런스 |
| import해서 바로 사용 | 초기 데이터 seeding용 |

---

## 회원가입 시 수집 필드 (01_patient_profile.json 기준)

- 이름, 생년월일, 성별
- 키, 몸무게
- 만성질환, 복약 중인 약, 알레르기
- 연락처, 비상연락처
- 주 병원/담당의
