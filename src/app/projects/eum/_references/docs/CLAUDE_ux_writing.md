# Eum (이음) — UX Writing Guide

> NHS Service Manual Content Guide 기반. 한국 의료 앱 맥락에 맞게 적용.
> 와이어프레임/프로토타입의 모든 텍스트 작성 시 이 가이드를 참조한다.
> 참고: https://service-manual.nhs.uk/content

---

## 핵심 원칙 (Core Principles)

| 원칙 | 설명 | 예시 |
|---|---|---|
| 정확함 (Accurate) | 임상 정보는 반드시 검증. 비유/은유 금지. 주관적 표현 지양 | "심장이 뛴다" X → "심박수 상승" O |
| 명확함 (Clear) | 쉬운 말 우선. 전문용어 사용 시 "(쉬운말/의학용어)" 형식 | "치질(치핵)" |
| 간결함 (Concise) | 문장 20단어 이내. 단락 3문장 이내. 능동태 사용 | "검사가 진행됩니다" X → "검사를 진행합니다" O |

---

## Voice & Tone

- **기본 톤**: 중립적·사실 기반 (neutral, factual)
- **태도**: 권위 있되 차분 (authoritative yet calm)
- **방향**: 권한부여형. 지시형 X (empowering, not patronizing)

### 표현 규칙

| 지양 | 권장 |
|---|---|
| "~해야 합니다" | "~할 수 있습니다", "~하는 것이 좋습니다" |
| "당신" | 직접 호칭 없이 서술, 또는 맥락에 따라 "환자분", "선생님" |

### 맥락별 톤 조절

| 맥락 | 톤 |
|---|---|
| 진단/처방 관련 | 진지하고 안심감을 주는 톤 |
| 생활습관/일상 건강 | 격려와 대화체 |
| 폼/입력 화면 | 대화하듯 자연스럽게 |

## 건강 문해력 (Health Literacy)

> 40% 이상의 성인이 건강 콘텐츠 이해에 어려움을 겪는다. 모든 텍스트는 이를 전제로 작성한다.

### 의학 용어

- 쉬운 말 먼저, 의학 용어는 괄호 안에 표기
- 예: "치질(치핵)", "혈압(Blood Pressure)"

### 숫자/통계

- "7명 중 1명" 형태 선호
- 퍼센트(%)와 자연빈도("N명 중 N명") 혼용 금지 — 한 가지만 사용
- 구체적 표현 사용: "오랫동안" X → "약 2주간" O

### AI 쉬운말 변환

- AI가 의학용어를 쉬운말로 변환할 때에도 위 규칙을 동일하게 적용
- 반드시 "AI가 변환한 내용"임을 명시

## 포맷 규칙 (Formatting)

### 제목

- 문장형 대소문자 (Sentence case)
- 65자 이내
- 키워드 앞배치
- 질문형 제목 금지 (질문 페이지 제외)

### 목록

- **글머리 기호 목록**: 소문자 시작, 마침표 없음, 항목 6개 이내
- **번호 목록**: 순서가 있는 절차에만 사용

### 강조

| 방법 | 사용 기준 |
|---|---|
| **굵은 글씨** | UI 요소를 지시할 때만 사용 (예: "**확인** 버튼을 누르세요") |
| *이탤릭* | 사용 금지 (링크와 혼동) |
| 밑줄 | 사용 금지 (링크와 혼동) |
| 강조 대안 | 문장 앞배치 또는 "중요:" 접두사 사용 |

### 링크

- "여기를 클릭" 금지 → 목적지를 설명하는 텍스트 사용
- "더 보기", "자세히" 단독 사용 금지 → 대상 명시 (예: "증상 기록 방법 자세히 보기")

## 숫자·날짜·시간

| 항목 | 규칙 | 예시 |
|---|---|---|
| 숫자 | 아라비아 숫자 사용 | 1, 2, 3 (일, 이, 삼 X) |
| 날짜 | "YYYY년 M월 D일" 또는 "YYYY.MM.DD" | 2026년 3월 8일, 2026.03.08 |
| 시간 | 12시간제, "오전/오후" 표기 | 오후 5시, 오후 5시 30분 (17:00 X) |
| 측정값 | 숫자+단위 붙여쓰기 | 250mg, 120/80mmHg, 36.5°C |
| 범위 | "-" 대신 "~" 또는 "부터 ~ 까지" | 3~5일, 10mg부터 20mg까지 |

## 폼 질문 작성

- 모든 질문의 **필요성을 먼저 검증** — 불필요한 질문은 제거
- 대화하듯 **자연스러운 질문 순서** 배치
- 민감한 질문은 **맥락 설명 제공** (예: "정확한 약물 상호작용 확인을 위해 필요합니다")
- **에러 메시지**: 무엇이 잘못되었는지 + 어떻게 해결하는지 함께 제시
  - X: "입력 오류"
  - O: "전화번호 형식이 올바르지 않습니다. 010-0000-0000 형식으로 입력해 주세요."
- **placeholder**: 예시 형태로 작성 (예: "예: 페니실린")

## 포용적 언어 (Inclusive Language)

| 영역 | 규칙 | 예시 |
|---|---|---|
| 연령 | 나이 기반 일반화 금지 | "노인은 ~하기 어렵다" X |
| 장애/질환 | 사람 중심 표현 | "장애인" → "장애가 있는 분" |
| 성별 | 필요 없는 경우 성별 특정 표현 지양 | "간호사 언니" X → "담당 간호사" O |
| 피부 증상 | 다양한 피부 톤을 고려한 표현 | "빨갛게 부어오름" → "부어오르며 색이 변함" |

## 의료 콘텐츠 품질 기준

- 임상 정보는 반드시 **임상 전문가 검토/승인** 필요
- **근거 기반 콘텐츠** — 출처 명시
- **최종 검토일** 표시 (예: "마지막 검토: 2026년 3월")
- AI 생성 콘텐츠: 반드시 "AI가 변환한 내용" 출처 명시 + 오류 가능성 안내
- **WCAG 2.2 AA** 접근성 준수

## Eum 프로젝트 특수 규칙

> 아래 규칙은 CLAUDE_legal_compliance.md, CLAUDE_data_hierarchy.md와 연동된다. 충돌 시 법적 제약이 우선한다.

### AI 출력 텍스트

- "진단", "확진" 표현 **절대 금지** → "참고 키워드", "관련 정보" 사용
- AI 출력에는 항상 면책 고지 동반

### 면책 고지

- 닫기(dismiss) 불가, 영구 노출
- 3가지 유형의 AI 경고 (CLAUDE_legal_compliance.md 참조)

### 환자용 AI 요약 (P-020)

- 의학용어(쉬운말) 병기 형식
- 예: "고혈압(혈압이 높은 상태)"

### 의사용 패널 (D-000, D-001)

- 320–360px 내에서 읽을 수 있는 간결한 텍스트
- 약어 허용 (의료진 대상이므로), 단 첫 등장 시 풀네임 병기

### 증상 강도 표현

- NRS 숫자 + 이모지 + 색상 **3중 인코딩** (CLAUDE_data_hierarchy.md 접근성 규칙 준수)
- 색상만으로 정보를 전달하지 않음
